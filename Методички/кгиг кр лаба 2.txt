растровый и векторный; растеризация, векторизация
растровая визуализация основана на представлении изображения в виде массива пикселов - bitmap
растр - массив (матрица) пикселов
битовая карта - цветовая глубина - одинаковое кол-во битов на кодировку каждого пиксела
типы растра: квадратный, прямоугольный, гексагональный (от положения пикселов в пр-ве)
разрешающая способность - расстояние между соседними пикселами - dpi
шаг растра может быть не равен размеру пикселов; шаг = 1 / dpi
размер растра - кол-во пикселов по осям
форма пикселов: квадратная с размером равным шагу растра, круглая с размером больше\меньше шага

классификация по глубине цвета:
1. двухцветные (1 бит на пиксел, черный\белый)
2. полутоновые (1 байт на пиксел, градации серого, 256 значений)
3. цветные (2+байт на пиксел, highColor - 2 байта, trueColor - 3 байта на пиксел[16 млн. цветов])
   srgb - 37% cie 1931
   adobe rgb - 45% cie 1931 (130% srgb)

колориметрия - наука, изучающая цвет и его измерения
один из базовых законов колориметрии - закон смешивания цветов (любой цвет - комбинация rgb)
законы смешивания цветов:
1. трехмерности - любой цвет можно представить линейной комбинацией трех цветов (Rk1 + Gk2 + Bk3)
2. непрерывности - к любому цвету можно подобрать бесконечно близкий
3. аддитивности - цвет зависит только от его компонент
линейная независимость: из двух базовых цветов нельзя получить третий

RGB: компонентный способ кодирования; излучающие у-ва (мониторы, телевизоры)
R = G = B = 0...255 (255^3 = 16,7млн)
WinAPI - 4 байта ( 00000000 bbbbbbbb gggggggg rrrrrrrr )

CMYK: субстрактивная модель кодирования; поглощающие у-ва (принтеры, плоттеры)
C = tR = G + B
M = tG = R + B
Y = tB = R + G
K = Black
на окрашенные пов-сти падает белый свет (сумма всех), и надо вычесть все, кроме того, что будет
отражаться (что мы будем видеть)
палитра - ограничение глубины цветов для конкретного изображения

графический формат - структура (способ) хранения инфы в файле
1. растровый - bmp, jpg - быстрые, но плохой масштаб
2. векторный - autocad - чертежи или текст, плоттеры заточены только под векторные изображения
3. метафайловый - coreldraw .cdr - вектор + растр

Для вывода изображения:
HANDLE (из файла) -> BITMAP (карта битов) -> HDC dest вывод в окно с помощью BitBlt())
StretchBlt(): добавляются widthSrc, widthDest - ширина исходного и выводимого изображения

DIB .bmp:
1. BITMAPFILEHEADER - описание типа файла, его размера и смещение по битам до начала кода картинки
2. BITMAPINFOHEADER - описание изображение, размера, типа компрессии, кол-во бит на пиксел и т.д.
3. RGBQUAD - палитра (таблица цветов) [R, G, B]		[часто отсутствует, напр. при 24-бит]
4. растр изображения в виде байтового массива пикселов, последователньо записываются байты
   строк растра и выравниваются до DWORD - 4 байт, если битов не хватает до этого размера
   (это смещение хранится в BITMAPFILEHEADER)

LoadImage() - для загрузки битмапа в ОЗУ и получения дескриптора для вывода на экран 
ShowBitMap() - вывод в окно [HWND - дескриптор окна, HBITMAP - дескриптор рисунка]
ClientToBMP() - сохранение файла
